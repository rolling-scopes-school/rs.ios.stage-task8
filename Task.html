Требования

  • Минимальная поддерживаемая версия операционной системы: iOS 12.1
  • В готовом приложении должно полностью воспроизводиться поведение, представленное в демонстрационном ролике
  • Основной язык реализации приложения: Objective-C. Для реализации экрана Drawings и вспомогательного отображения Timer необходимо использовать язык Swift
  • Для тестирования интерфейса и логики работы приложения необходимо использовать iPhone SE 2nd generation с установленной операционной системой iOS 13.5
  • Дизайн готового приложения на устройстве iPhone SE 2nd generation должен в точности соответствовать описанному в файле дизайна
  • Название приложения: "RSSchool_T8"
  • Приложение работает только со светлой темой
  • Для реализации UI используется исключительно UIKit

Критерии

  • Пользователь может (должен иметь возможность, используя готовое приложение) отрисовать изображение по каждому шаблону: tree, landscape, head, planet. Готовое изображение максимально точно повторяет описанное в файле дизайна (допустимы малые неточности в местах). +15 баллов
  • Пользователь может регулировать время отрисовки. Время отрисовки соответствует заданному пользователем. Например, корректным будет случай, когда при настройке длительности отрисовки, равной 5 секундам, отрисовка будет длиться 5 секунд (с точностью до 0.3 секунды). Некорректным будет поведение, когда пользователь задал время отрисовки, равное, например, 3 секундам, а следующий цикл отрисовки длился 4 секунды. +15 баллов
  • Пользователь может выбирать цвета для отрисовки. Максимальное количество: 3 цвета, минимальное количество: 0 цветов. При отрисовке обязательно используются либо заданные пользователем цвета, либо цвета по умолчанию. Пользователь может добавлять цвета в выбранные и удалять их из выбранных. Используемые в отрисовке цвета соответствуют выбранным. Если пользователь выбрал и сохранил цвета красный, жёлтый и cиний, тогда только эти цвета будут использованы в отрисовке. Например, некорректным считается поведение, когда пользователь выбрал и сохранил для отрисовки цвета красный, жёлтый и cиний, а после этого в следующем цикле отрисовке были использованы цвета красный, жёлтый и розовый. Корректным будет поведением, когда пользователь выбрал и сохранил цвета, например, красный, жёлтый и cиний, а затем в следующем цикле отрисовки были использованы цвета красный, жёлтый и cиний. +15 баллов
  • Дизайн готового приложения на устройстве iPhone SE 2nd generation с версией iOS 13.5 в точности соотвествует описанному в файле дизайна. +15 баллов
  • В готовом приложении на устройстве iPhone SE 2nd generation с версией iOS 13.5 можно воспроизвести flow из демонстрационного ролика. +10 баллов
  • Пользователь может экспортировать готовое изображение в приложения Photos и Files в формате PNG или JPEG (❗️). +10 баллов
  • В ходе работы приложения не наблюдаются утечки памяти. +10 баллов
  • Был использован язык Swift для реализации:
    • экрана Drawings
    • вспомогательного отображения Timer

    Использование языка Swift предполагает реализацию с его помощью только основных элементов UI и объектов бизнес-логики, с которыми у экрана Drawings или у вспомогательного отображения Timer происходит прямое взаимодействие. Всё, что также описано на языке Swift, но не имеет прямого отношения к данным элементам, должно быть рассмотрено как нарушение данного критерия.

    То есть, например, во вспомогательном отображении Timer нужно создать объект типа NSTimer (или типа Timer (который часть Foundation)) и с помощью совместимости с objc связать эту логику с основным экраном Artist. Логику кнопки Reset нужно, обратно, связать с элементом Timer.

    В реализации готового приложения необходимо задействовать механизм совместимости языков Swift и Objective-C. Если экран Drawings и вспомогательное отображение Timer были реализованы на языке Objective-C, тогда данный пункт не засчитывается и не рассматривается как нарушение.

    +10 баллов

  • Язык Swift был использован где-либо ещё, кроме экрана Drawings и вспомогательного отображения Timer. Данный пункт - это нарушения из пункта выше. -10 баллов
  • Для отрисовки изображения по шаблону не был использован объект NSTimer либо для отрисовки изображения были задействованы методы и типы для явной анимации (методы UIView animateWithDuration:... transitionWithView:..., тип UIViewPropertyAnimator, тип CAAnimation и его наследники). Этот пункт не касается типа CALayer, его наследников и их внутреннего поведения. 🎨 🍪 🥚 -10 баллов
  • Объекты NSTimer продолжают работать тогда, когда были замещены другими объектами NSTimer или когда время их работы было завершено. Например, неправильным будет считаться случай, когда после окончания цикла отрисовки соответствующий этому циклу объект NSTimer продолжает работать. -5 баллов
  • Приложение не запускается или элементы UI не отображаются так, как это описано в файле дизайна, или невозможно воспроизвести flow из демонстрационного ролика на устройстве iPhone 8 с версией ниже 13.0 и выше 12.1 (имеется ввиду любая версия меньше 13.0 и, одновременно, выше 12.1). -5 баллов
  • Другие мелкие неточности. Этот критерий отдаётся на волю проверяющего и может включать в себя какие-либо замечания по коду, реализации UI и комплектации проекта. Замечания проверяющего не должны пересекаться с требованиями, пунктами и примерами, описанными в данном файле, в файлах дизайна и представленными в демонстрационном ролике. Оценка за данный пункт должна быть выставлена только один раз и только когда все остальные пункты из блока критериев были проверены и оценены. Причина выставления данного пункта должна быть максимально объективной и подкреплена примером из проверяемого проекта. Проверяющий также должен предоставить свой вариант решения поставленного им замечания. Человек, который получил данное замечание, но не получил достаточный фидбек, требуемый в этом пункте, вправе требовать уточнения по этому критерию у проверяющего. -5 баллов

Дополнение к критериям

Если в приложении была использована анимация (например, для анимирования элементов на вспомогательном отображении Palette или анимация тени у кнопок), это не должно приниматься за ошибку и несоответствие дизайну. Исключением для использования анимации является пункт, описанный в основном блоке критериев.

На усмотрение оценивающего, к итоговой оценке можно добавить баллы. Это можно делать исходя из наличия и качества анимированных элементов (беря во внимание пункт из основного блока критериев, а также за общее впечатление от проверяемого проекта.

Последнее никак не должно пересекаться с требованиями, пунктами и примерами, описанными в данном файле, в файлах дизайна и представленными в демонстрационном ролике. Например, это может быть общее положительное впечатление от качества кода или применённые знания о языке (к примеру, использование категорий для типа UIColor). Максимальное значение: +10 баллов, минимальное: +0 баллов.

Эту оценку можно добавить в том случае, если после оценивания всей работы конечная сумма баллов меньше 100. При этом после добавления этой оценки сумма баллов не должна превышать 100.

Эту оценку нельзя добавить тогда, когда не все критерии из основого блока были проверены и оценены (включая пункт о мелких неточностях).

Например, итоговая сумма получилась 95, при этом в приложении была реализована анимация. Оценивающий решил добавить оценку за эти анимации. Максимальное значение, которое он может добавить - +5 баллов, чтобы в итоге было 95 + 5 = 100.

Демонстрационный ролик показывает только flow, который необходимо реализовать и протестировать в готовом приложении. Для уточнения цветов, размеров и др. свойств элементов интерфейса смотрите файл дизайна.

Описание

В этом задании необходимо реализовать приложение-рисовалку. Рисование происходит по заранее заданному шаблону, процесс производится пошагово на отображении.

Всего шаблонов 4:

  • tree - находится в tree-frame
  • head - находится в head-frame
  • landscape - находится в landscape-frame
  • planet - находится в planet-frame

Для перевода в CGPath необходимо каждый из шаблонов экспортировать в формат SVG, после чего открыть этот файл в текстовом редакторе (например, Visual Studio Code) и извлечь последовательность, описывающую каждый из путей.

Полученный путь представлен на языке Swift и содержит некоторое количество неточностей. Данные неточности необходимо исправить, а путь описать на языке Objective-C.

Экран запуска приложения представлен в файле дизайна в элементе launchscreen.

Основной экран - Artist. На нём находится отображение Canvas, на котором происходит отрисовка, и 5 кнопок: Open Palette, Open Timer, Draw, Share, Drawings.

  • Заголовок экрана - "Artist"

Этот экран может находиться в одном из трёх состояний:

  • idle
  • draw
  • done

В состоянии idle активны кнопки Open Palette, Open Timer, Draw и Drawings, кнопка Share неактивна. Canvas не отображает рисунок.

При нажатии на кнопку Draw на отображении Canvas отрисовывается заданный шаблон. Экран Artist переходит в состояние draw. В этом состоянии кнопки Open Palette, Draw, Open Timer и Share неактивны, кнопка Drawings активна.

Отрисовка происходит с помощью трёх путей. Цвета для путей извлекаются из вспомогательного отображения Palette, время отрисовки - из вспомогательного отображения Timer. Обратите внимание на то, что концы у этих путей в процессе отрисовки и в готовом рисунке должны быть скруглены. Прогресс отрисовки должен отображаться на Canvas.

По умолчанию отрисовка происходит со следующими параметрами:

  • Время отрисовки: 1 секунда
  • Цвет: #000000

где TTT﻿ - время отрисовки.

После окончания отрисовки экран Artist переходит в состояние done. В этом состоянии кнопки Open Palette и Open Timer неактивны, кнопки Drawings, Reset и Share активны. На Canvas отображается рисунок.

При нажатии на кнопку Reset экран Artist переходит в состояние idle.

Переход из состояния done в состояние idle должен занимать не более 1 секунды.

При нажатии на кнопку Share отображается системное окно, с помощью которого пользователь может экспортировать изображение из отображения Canvas. Изображение должно включать в себя только результат работы отрисовки (без границы отображения Canvas). Изображение должно быть в формате PNG или JPEG (❗️).

Пользователь должен иметь возможность сохранять и просматривать экспортируемое изображение в приложении Photos и приложении Files.

Приложение должно запрашивать доступ к пользовательской галерее фотографий. Для этого необходимо в файле Info.plist указать ключ NSPhotoLibraryAddUsageDescription со следующим содержанием:

  • "This application needs access to your library to save the image"

Если пользователь отказался предоставлять доступ, это разрешение не должно запрашиваться снова.

Разрешение должно запрашиваться только при первой попытке доступа к галерее.

При нажатии на кнопку Open Palette открывается вспомогательное отображение Palette с палитрой цветов. На Palette находятся 12 кнопок, каждой из которых присвоен отдельный цвет. Все 12 кнопок активны. Цвета кнопок идут в соответствии с файлом дизайна приложения.

Пользователь может выбрать три цвета. Выбор происходит в порядке очереди: если пользователь уже выбрал три цвета и выбирает ещё один, тогда первый уже выбранный перестаёт быть выбранным.

Пример: выбраны цвета в следующей последовательности красный, синий, жёлтый. Пользователь выбирает следующий цвет - фиолетовый. Цвет красный перестаёт быть выбранным и последовательность выбранных цветов становится следующей: синий, жёлтый, фиолетовый.

Пользователь может исключать уже выбранные цвета повторным нажатием.

При выборе цвета фоновый цвет вспомогательного отображения Palette меняется на выбранный цвет. Через 1 секунду после выбора фоновый цвет меняется на исходный, указанный в дизайне. Если в течение этой секунды выбирается новый цвет, тогда отсчёт начинается заново для нового выбранного цвета.

Если в течение секунды выбранный цвет был исключён из списка выбранных, отсчёт для текущего цвета не должен останавливаться в момент исключения.

Например, был выбран цвет зелёный. Фоновый цвет был установлен в зелёный и начался отсчёт 1 секунды. Через 0.4 секунды после выбора цвет зелёный был исключён из списка выбранных. При этом фоновый цвет ещё 0.6 секунд будет равен зелёный (если после этого не был выбран новый цвет, как описано выше).

При исключении цвета фоновый цвет вспомогательного отображения Canvas не меняется на исключаемый цвет.

При нажатии на кнопку Save выбранные цвета сохраняются для использования в следующем цикле отрисовки в отображении Canvas, вспомогательное отображение Palette скрывается с экрана. Если количество выбранных цветов меньше 3, тогда недостающие цвета дополняются цветом #000000, пока общее количество цветов выбранных цветов не будет равно 3. Итоговые цвета в случайном порядке присваиваются 3 путям, отрисовывающим шаблоны на Canvas.

Например, пользовател выбрал и сохранил цвета красный и синий, тогда после сохранения эти цвета дополнятся цветом #000000, чтобы их общее количество было равно 3. Итоговый набор будет следующим: красный, синий, #000000. После этого цвета в случайном порядке присвоятся путям для отрисовки на Canvas.

Если пользователь выбрал и сохранил цвета красный, синий и розовый, тогда набор не будет дополняться цветом #000000, так как количество выбранных цветов и так равно 3. Итоговый набор будет следующим: красный, синий и розовый.

Если пользователь выбрал и сохранил цвет красный, тогда дополнение цветом #000000 должно давать следующий итоговый набор: красный, #000000, #000000.

Если пользователь сохранил пустой набор выбранных цветов, тогда путям присваиваются цвета по умолчанию.

Порядок итогового набора цветов несущественен, так как цвета присваиваются путям в случайном порядке. Названия цветов (например, красный) во всём файле использованы только для описания принципа реализации. Цвета, необходимые для использования в готовом приложении, и их порядок описаны в файле дизайна.

При нажатии на кнопку Open Timer открывается вспомогательное окно Timer. На нём расположены интерактивный элемент Slider и три текстовых элемента. Элемент слева от Slider - min, справа - max, посередине - current. Элемент min имеет значение "1", max - "5", current - текущее значение из элемента Slider. Элемент Current отображает числовое значение с точностью до двух знаков после запятой.

По умолчанию элемент Slider имеет значение 1.00Пользователь может взаимодействовать с элементом Slider для увеличения или уменьшения значения элемента Slider. Минимальное значение элемента Slider - 1.00, максимальное - 5.00

При нажатии на кнопку Save значение из элемента Slider сохраняется для использования в следующем цикле отрисовки в отображении Canvas, вспомогательное отображение Palette скрывается с экрана.

При нажатии на кнопку Drawings открывается экран Drawings, на котором расположены 4 кнопки: Planet, Head, Tree, Landscape, а также кнопка возврата на предыдущий экран Artist. Заголовок - "Drawings". Все кнопки на этом экране активны.

Каждой кнопке соотвествует свой шаблон:

  • Кнопка Planet - шаблон planet
  • Кнопка Head - шаблон head
  • Кнопка Tree - шаблон tree
  • Кнопка Landscape - шаблон landscape

При нажатии на кнопку соответствующий ей шаблон запоминается в системе для использования в следующем цикле отрисовки на Canvas.

Одна из кнопок находится в состоянии selected. Это состояние определяется тем, какой из шаблонов выбран в системе для отрисовки на отображении Canvas. По умолчанию это шаблон Head.
